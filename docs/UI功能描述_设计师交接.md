# ☕ 手冲方案卡 — UI 功能描述（设计师交接文档）

> 产品定位：面向手冲咖啡爱好者的小工具。用户在小红书/抖音看到感兴趣的手冲方案视频或图文，复制链接粘贴进来，AI 自动解析并输出一张结构化的「手冲方案参数卡」。

---

## 一、整体页面结构

单页应用，移动端优先（建议设计宽度 375pt），桌面端居中显示（最大宽度 640px）。

页面从上到下依次为：

1. **顶部导航区**
2. **输入区**
3. **进度指示区**（提交后显示）
4. **结果卡片区**（完成后显示）
5. **底部信息**
6. **设置弹窗**（点击设置图标触发）

---

## 二、各区域功能详解

### 2.1 顶部导航区

| 元素 | 说明 |
|------|------|
| Logo 图标 | 产品 Logo（当前用 ☕ 占位） |
| 产品名称 | 「手冲方案卡」 |
| Tagline | 一句话说明，如"粘贴小红书链接，AI 提取冲煮参数" |
| 设置按钮 | 右上角，点击打开 API 设置弹窗 |

### 2.2 输入区

一个卡片式容器，内含三行：

| 元素 | 类型 | 说明 |
|------|------|------|
| 链接输入框 | 多行文本 | 主输入。Placeholder: "粘贴小红书/抖音上手冲咖啡方案的链接..." |
| 分割线 | — | 视觉分隔 |
| 补充要求输入框 | 单行文本 | 可选。Placeholder: "补充要求（可选）— 如：重点关注水温和注水手法" |
| 分割线 | — | 视觉分隔 |
| 底部操作行 | 横排 | 包含：模型选择器 + 提取按钮 |

**模型选择器**（下拉菜单）：

| 分组 | 模型 |
|------|------|
| 推荐 | DeepSeek V3、DeepSeek R1 |
| 通用 | Qwen3 235B、Kimi K2、GPT-4o、Claude Sonnet 4 |

**提取按钮**：

| 状态 | 文案 | 样式 |
|------|------|------|
| 默认 | "提取方案" | 实心按钮 |
| 加载中 | "提取中..." | 置灰不可点 |

**快捷键**：`Cmd/Ctrl + Enter` 触发提交。

### 2.3 进度指示区

提交后显示，完成后隐藏。卡片内含标题 + 步骤列表。

| 步骤 | 文案 | 说明 |
|------|------|------|
| Step 1 | 解析链接 | 解析 URL、跟随重定向、抓取页面 |
| Step 2 | 提取音频 / 图文内容 | 视频→音频转写 / 图片→OCR |
| Step 3 | 聚合文本 | 将多源内容合并为统一文本 |
| Step 4 | AI 精准提取参数 | 第一轮 LLM：从内容中提取明确提到的参数 |
| Step 5 | AI 智能补全缺失 | 第二轮 LLM：推理补全未提及的参数 |

每个步骤有三种状态：
- **待处理**（灰色圆圈）
- **进行中**（高亮 + 脉冲动画）
- **已完成**（绿色 ✓）

### 2.4 错误提示区

当处理失败时显示（替代进度区），包含错误标题 + 错误描述文本。

### 2.5 结果卡片区 — 手冲方案参数卡 ⭐

这是产品的核心输出，需要重点设计。整张卡片从上到下分为以下区块：

#### ① 卡片头部

| 字段 | 说明 | 示例值 |
|------|------|--------|
| 来源平台 | 小红书 / 抖音 | "小红书" |
| 作者 | @作者名 | "· @Owen的咖啡日记" |
| 方案标题 | 通用化命名（AI 自动概括，非照搬原标题） | "浅烘日晒非洲豆 V60 高甜感方案" |
| 方案概述 | 一句话概括特点和适用豆子类型 | "适用于浅烘日晒处理的非洲豆，通过分段注水突出甜感和果汁感" |

#### ② 冲煮参数网格

6 个核心参数，3列 × 2行 网格布局（移动端可 2列 × 3行）：

| 参数 | 字段名 | 示例值 |
|------|--------|--------|
| 粉量 | dose | 15g |
| 水量 | water_amount | 225ml |
| 粉水比 | ratio | 1:15 |
| 水温 | water_temp | 92°C |
| 研磨度 | grind_size | 中细 / C40 24格 |
| 萃取时间 | total_time | 2分15秒 |

每个格子包含：**大字数值** + 小字标签。

#### ③ 咖啡豆 & 器具信息

以标签（Tag/Chip）形式横排展示，每个标签包含"类别名 + 值"：

| 类别 | 字段名 | 示例值 |
|------|--------|--------|
| 滤杯 | dripper | V60 |
| 产地 | origin | 埃塞俄比亚 |
| 烘焙 | roast_level | 浅烘 |
| 处理法 | process | 日晒 |
| 豆子 | bean_name | i-SIDE 比亚日晒 |
| 滤纸 | filter_paper | Hario V60纸滤杯（白色） |
| 器具备注 | equipment_notes | 百胜图E6Air磨豆机 |
| 豆子备注 | bean_notes | （如有） |

> 没有值的标签不显示。这个区块仅当至少有一个标签有值时才显示。

#### ④ 注水步骤（时间线）

按时间顺序展示每段注水，每一步包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| time | 左侧时间列 | "30s" / "0:30-1:00" |
| stage | 阶段名（加粗） | "闷蒸" / "推粉" / "高甜水位" |
| technique | 手法描述 | "中心点注水至产生泡泡后开始绕圈" |
| water_ml | 水量标签 | "注至40ml" / "注至120ml" |

布局建议：左列固定宽度放时间，右列放阶段名+手法+水量标签。步骤之间用虚线分隔。

> 没有注水步骤时，此区块隐藏。

#### ⑤ 风味描述 & 冲煮技巧

| 区块 | 说明 | 示例 |
|------|------|------|
| 风味描述 | 斜体文本 | "甜感明显、香气表现佳、口感如果汁般顺滑" |
| 冲煮技巧 | 引用/卡片样式，带左边线 | "推粉阶段帮助提升甜感物质萃取，第三段保持注水与滴滤速度一致" |

> 两者都没有时，整个区块隐藏。

#### ⑥ AI 推理标记 ⭐⭐

**这是一个重要的设计特性。**

部分参数可能原内容中没有提到，由 AI 根据已知信息推理补全。需要在视觉上区分：

**哪些字段可能被 AI 补全**：
- 冲煮参数网格中的 6 个参数
- 咖啡豆 & 器具标签中的所有标签
- 风味描述
- 注水步骤中的时间

**标记方式**（当前方案，设计师可重新设计）：

| 组件 | 当前标记方式 |
|------|-------------|
| 参数格子 | 右上角小标签 "AI推理"，数值用不同颜色 |
| 信息标签 | 虚线边框 + 后缀 "·AI" |
| 步骤时间 | 时间右上方小号 "AI" 上标 |
| 风味描述 | 区块标题后加 "· AI推理" 标记 |

**图例说明**：当卡片中有任何被 AI 补全的字段时，在卡片底部显示一行图例说明文字。

#### ⑦ 卡片底部

| 元素 | 说明 |
|------|------|
| 查看原文链接 | 点击跳转到原始小红书/抖音内容 |
| 模型名称标签 | 显示使用了哪个 AI 模型（如 "DeepSeek V3"） |
| 复制方案按钮 | 点击复制为纯文本格式，按钮文字短暂变为"已复制" |

**复制的文本格式示例**：

```
☕ 浅烘日晒非洲豆 V60 高甜感方案
适用于浅烘日晒处理的非洲豆，通过分段注水突出甜感和果汁感

📋 冲煮参数
粉量: 20g  水量: 300g  粉水比: 1:15
水温: 92°C  研磨: 白砂糖偏细  时间: 2分15秒 [AI推理]

滤杯: V60滤杯  产地: 埃塞俄比亚  烘焙: 浅烘

💧 注水步骤
30s 闷蒸 注至40ml 中心点停留2秒后绕圈注水
0:30-1:00 推粉 注至120ml 中心点注水至产生泡泡后开始绕圈
1:00-1:45 高甜水位 注至300ml 大流量拉高水位至粉层高度

🍊 风味: 柑橘、莓果、热带水果 [AI推理]
💡 技巧: 推粉阶段帮助提升甜感物质萃取

原文: https://www.xiaohongshu.com/...
```

> 被 AI 补全的字段在复制文本中会带 `[AI推理]` 标记。

### 2.6 设置弹窗（Modal）

点击右上角设置图标打开。包含：

| 元素 | 说明 |
|------|------|
| 标题 | "API 设置" |
| 说明文字 | 引导用户填入 JieKou AI 的 API Key |
| 状态指示 | 绿色"已配置 — sk_Lcr***xxxx" 或 红色"未配置 API Key" |
| API Key 输入框 | 密码类型，保存后不显示完整值 |
| Base URL 输入框 | 默认填 JieKou AI 的地址，可改 |
| 保存按钮 | 保存后显示 Toast 提示 "配置已保存" |

**自动弹出逻辑**：首次加载如果检测到 API Key 未配置，延迟 500ms 自动打开此弹窗。

### 2.7 Toast 通知

固定在屏幕底部居中，短暂显示后自动消失（2秒），用于反馈操作结果。

## 三、完整数据结构（BrewCard）

后端返回给前端的 JSON 数据结构：

```json
{
  "title": "浅烘日晒非洲豆 V60 高甜感方案",
  "summary": "适用于浅烘日晒处理的非洲豆，通过分段注水突出甜感和果汁感",

  "bean_name": "i-SIDE 比亚日晒",
  "origin": "埃塞俄比亚",
  "roast_level": "浅烘",
  "process": "日晒",
  "bean_notes": "",

  "grind_size": "白砂糖偏细（百胜图E6Air磨豆机9档）",

  "dose": "20g",
  "water_amount": "300g",
  "ratio": "1:15",
  "water_temp": "92°C",
  "total_time": "2分15秒",

  "dripper": "V60滤杯",
  "filter_paper": "Hario V60纸滤杯（白色）",
  "equipment_notes": "使用百胜图E6Air磨豆机，研磨度9档",

  "pour_steps": [
    {
      "stage": "闷蒸",
      "water_ml": "注至40ml",
      "time": "30s",
      "technique": "中心点停留2秒后绕圈注水",
      "_inferred": []
    },
    {
      "stage": "推粉",
      "water_ml": "注至120ml",
      "time": "0:30-1:00",
      "technique": "中心点注水至产生泡泡后开始绕圈，由外向内回中心",
      "_inferred": ["time"]
    },
    {
      "stage": "高甜水位",
      "water_ml": "注至300ml",
      "time": "1:00-1:45",
      "technique": "大流量拉高水位至粉层高度，再回到中心点小流量定点注水",
      "_inferred": ["time"]
    }
  ],

  "flavor_notes": "柑橘、莓果、热带水果",
  "tips": "推粉阶段帮助提升甜感物质萃取，第三段保持注水与滴滤速度一致，避免过度扰流带来苦涩",

  "source_platform": "xiaohongshu",
  "source_url": "https://www.xiaohongshu.com/explore/..."

```

**关键字段说明**：
- `inferred_fields`：数组，列出哪些顶层字段是 AI 推理补全的
- `pour_steps[].\_inferred`：数组，列出该步骤中哪些子字段是 AI 推理补全的
- 值为空字符串 `""` 的字段：该信息不存在，对应 UI 组件不显示

## 四、交互状态流转

```
[空闲] → 用户粘贴链接 → 点击提取 → [加载中]
  ↓
显示进度步骤（5步依次推进）
  ↓
成功 → [显示方案卡片]（进度区隐藏）
失败 → [显示错误提示]（进度区隐藏）
```

用户可以随时再次粘贴新链接重新提取，之前的结果会被替换。

## 五、设计要点建议

1. **卡片感**：结果卡片是核心输出物，应有高级感、可截图分享
2. **AI 推理标记**：需要在「不破坏视觉整体性」的前提下，让用户能区分哪些是原文提取、哪些是 AI 补全
3. **参数网格**：6 个核心参数是最关键的信息，需要视觉权重最高
4. **注水步骤时间线**：手冲爱好者非常关注每一段注水的时间和手法，建议设计得清晰易读
5. **移动端体验**：主要使用场景是用户在手机上看到小红书内容后切过来用，移动端优先
6. **咖啡调性**：整体视觉风格建议暖色调、自然质感，契合手冲咖啡的精品感
